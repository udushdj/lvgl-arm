# LVGL医疗叫号系统卡死问题 — 排查与修复总结

## 现象

在ARM嵌入式Linux开发板（GEC6818）上，点击Screen1的"患者详情"按钮切换到Screen2时UI卡死。不连接服务器（患者队列为空）时不卡，连接服务器（10个患者）后卡死。多次来回切换后必然复现。

## 排查过程

| 尝试 | 假设 | 结论 |
|------|------|------|
| 1 | HTTP同步socket阻塞主循环 | 部分原因，改为全程非阻塞socket |
| 2 | `lv_scr_load_anim`动画与widget创建冲突 | 部分原因，改为`LV_EVENT_SCREEN_LOADED`事件触发刷新 |
| 3 | 中文字体`my_font_cn`线性查找太慢 | **排除**，LVGL使用`_lv_utils_bsearch`二分查找 |
| 4 | 换成`LV_FONT_DEFAULT`英文字体 | **排除**，仍然卡死 |
| 5 | `lv_obj_clean`有bug | **排除**，LVGL 8.3源码正确 |
| 6 | 内存耗尽 | **根因确认** |

## 根因

**`LV_MEM_SIZE = 48KB`，LVGL的内部内存池太小。**

LVGL所有控件对象（lv_obj_t）、样式属性、flex布局数据、事件回调、动画、文本缓存等都在这个48KB的静态池中分配。

### 内存消耗计算

```

│ 项目                         │ 大小         │ 位置                      │
├─────────────────────────────┼──────────────┼──────────────────────────┤
│ 显示缓冲 buf[1536000]        │ 6 MB (Bug!)  │ BSS段                    │
│ bg_screen1 (ARGB8888 800×480)│ 1.5 MB       │ rodata                   │
│ bg_screen2 (ARGB8888 800×480)│ 1.5 MB       │ rodata                   │

│ my_font_cn_48 位图 (~200字)  │ ~225 KB      │ rodata                   │
│ LVGL对象池 (LV_MEM_SIZE)     │ 48 KB ❌     │ BSS段                    │
│ 代码段 + LVGL库              │ ~500 KB      │ text段                   │
│ 栈                           │ 8 MB         │ 虚拟地址                  │
│ 杂项 (sync_buf/cJSON等)      │ ~200 KB      │ BSS/data                 │
├─────────────────────────────┼──────────────┼──────────────────────────┤
│ 总计                         │ ~20 MB       │                          │
└─────────────────────────────┴──────────────┴──────────────────────────┘
```

### 关键内存细节

单个患者行的LVGL对象消耗：
```
外层btn (~200B) + 患者label (~150B) + 置顶btn (~200B) + pin_label (~80B)
+ 样式属性 (~300B) + 事件回调 (~20B) = ~950 字节/患者
```

Screen2患者列表：10患者 × 950B ≈ **10KB**（仅患者行）

全部4个屏幕预创建约105个控件，占用约 **25KB**。

每次`refresh_patient_list`调用`lv_obj_clean`释放旧控件再创建新控件。48KB池经过几次分配-释放周期后碎片化，新的大块分配失败 → `lv_mem_alloc`返回NULL → LVGL内部未处理 → **卡死**。

### 附带Bug：显示缓冲浪费

`main.c`中`DISP_BUF_SIZE 800*480*4 = 1,536,000`像素，实际屏幕仅384,000像素。4倍浪费，多占用 **4.5MB**。

## 修复

### 1. 增大LVGL内存池 (`lv_conf.h:52`)

```c
// 改前
#define LV_MEM_SIZE (48U * 1024U)    // 48KB — 不够

// 改后
#define LV_MEM_SIZE (256U * 1024U)   // 256KB — 充足
```

### 2. 修复显示缓冲大小 (`main.c:47`)

```c
// 改前
#define DISP_BUF_SIZE 800*480*4      // 1,536,000像素 = 6MB

// 改后
#define DISP_BUF_SIZE (800*480)      // 384,000像素 = 1.5MB
```

### 3. 同步socket全程非阻塞 (`sync_manager.c`)

重写状态机：socket始终`O_NONBLOCK`，使用`poll()`代替`select()`，`send()`/`recv()`返回`EAGAIN`时安全退出。

### 4. 屏幕切换时延迟刷新 (`ui.c`)

所有导航按钮（患者详情、返回）只做屏幕切换，widget刷新通过`LV_EVENT_SCREEN_LOADED`在动画完成后触发，避免动画渲染期间创建大量widget。

### 5. 附带修复

| 文件 | 问题 | 修复 |
|------|------|------|
| `linked_list.c` | `kill_list_node`递归释放，长链表栈溢出 | 改为迭代 |
| `ui_helpers.c` | `_ui_screen_delete`空指针判断反了 | 修正条件 |

## 最终内存对比

```
修复前:
  LV_MEM_SIZE:    48 KB → 碎片化导致分配失败
  显示缓冲:       6 MB  → 浪费4.5MB

修复后:
  LV_MEM_SIZE:   256 KB → 充足余量
  显示缓冲:     1.5 MB → 精确匹配屏幕分辨率
```

净节省物理内存约 **4.3MB**，LVGL内部池容量增加 **5.3倍**。
