# 屏幕闪烁功能实施计划

## 需求分析

用户希望在 `get_cur_patient_id_name1` 函数中对 `ui_Current0number0of0people0waiting0text0box`（等待人数标签）添加闪烁/闪烁效果，以提示患者已经发生切换。

## 调用链分析结果

| 患者切换操作 | 触发的UI刷新 |
|---|---|
| **程序启动** (`main.c`) | `get_cur_patient_id_name1` → `get_waiting_cur_number` → `get_next_patient_id_name` → `flush_panel_cur_date1` |
| **点击"下一位患者"** (`change_to_next_patient`) | 弹出队首 → 内联更新3个标签 → `flush_panel_cur_date1` |
| **从管理页返回** (`ui_event_Button1`) | 切屏 → `get_cur_patient_id_name1` → `get_waiting_cur_number` → `get_next_patient_id_name` → `flush_panel_cur_date1` |

**关键发现**：`change_to_next_patient` 没有调用 `get_cur_patient_id_name1`，而是内联复制了标签更新逻辑。因此我们需要在**两个位置**添加闪烁效果：
1. `get_cur_patient_id_name1` - 覆盖返回Screen1和点击标签的场景
2. `change_to_next_patient` - 覆盖点击"下一位患者"按钮的场景

## 技术方案

### 方案选择：LVGL定时器 + 透明度动画

使用 `lv_timer_create` 创建一个有限次数的闪烁定时器，通过循环修改 `ui_Current0number0of0people0waiting0text0box` 的透明度 (`lv_obj_set_style_opa`) 来实现平滑闪烁。

**为什么选这个方案：**
- 不需要隐藏对象（避免布局抖动）
- 透明度变化更平滑自然
- 定时器可以精确控制闪烁次数和停止
- 兼容 LVGL 8.3 的 API

### 详细实施步骤

#### 步骤1：在 `ui_events.c` 中添加闪烁定时器相关代码

需要添加：
- 闪烁定时器指针（static 变量）
- 闪烁计数器（控制闪烁次数）
- 定时器回调函数
- 启动闪烁的辅助函数

#### 步骤2：修改 `get_cur_patient_id_name1` 函数

在函数末尾（设置文本后）调用启动闪烁函数，触发 `ui_Current0number0of0people0waiting0text0box` 的闪烁动画。

**注意**：需要在 `get_cur_patient_id_name1` 中提前确保 `ui_Current0number0of0people0waiting0text0box` 已创建，否则闪烁对象为 NULL。

#### 步骤3：修改 `change_to_next_patient` 函数

在更新 `ui_Current0number0of0people0waiting0text0box` 文本后，同样调用启动闪烁函数。

### 闪烁参数设计

| 参数 | 值 | 说明 |
|---|---|---|
| 闪烁总次数 | 6（3个完整周期） | 闪烁3次（亮→暗→亮 为一个周期） |
| 定时器间隔 | 300ms | 每300ms切换一次透明度状态 |
| 正常透明度 | LV_OPA_COVER (255) | 完全不透明 |
| 闪烁透明度 | LV_OPA_30 (30) | 接近透明但可辨认 |
| 优先级 | LV_TASK_PRIO_MID | 中等优先级 |

### 涉及文件

| 文件 | 修改内容 |
|---|---|
| `UIManager/ui_events.c` | 添加闪烁定时器变量、回调函数、启动函数<br>修改 `get_cur_patient_id_name1` 和 `change_to_next_patient` |
| `UIManager/ui_events.h` | 无需修改（闪烁为内部实现） |

### 伪代码设计

```c
// ===== 新增：闪烁相关变量 =====
static lv_timer_t * blink_timer = NULL;
static int blink_count = 0;
#define BLINK_TOTAL_COUNT 6    // 闪烁6次（3个亮->暗周期）
#define BLINK_INTERVAL_MS 300  // 每300ms切换一次

// ===== 新增：闪烁定时器回调 =====
static void blink_timer_cb(lv_timer_t * timer)
{
    if (ui_Current0number0of0people0waiting0text0box == NULL) {
        // 对象不存在，停止闪烁
        lv_timer_del(blink_timer);
        blink_timer = NULL;
        return;
    }
    
    if (blink_count >= BLINK_TOTAL_COUNT) {
        // 闪烁结束，恢复完全透明度
        lv_obj_set_style_opa(ui_Current0number0of0people0waiting0text0box, LV_OPA_COVER, 0);
        lv_timer_del(blink_timer);
        blink_timer = NULL;
        return;
    }
    
    // 奇数次：变暗，偶数次：变亮
    if (blink_count % 2 == 0) {
        lv_obj_set_style_opa(ui_Current0number0of0people0waiting0text0box, LV_OPA_30, 0);
    } else {
        lv_obj_set_style_opa(ui_Current0number0of0people0waiting0text0box, LV_OPA_COVER, 0);
    }
    blink_count++;
}

// ===== 新增：启动闪烁 =====
static void start_blink(void)
{
    // 先停止可能正在运行的闪烁
    if (blink_timer != NULL) {
        lv_timer_del(blink_timer);
        blink_timer = NULL;
    }
    
    if (ui_Current0number0of0people0waiting0text0box == NULL) return;
    
    blink_count = 0;
    blink_timer = lv_timer_create(blink_timer_cb, BLINK_INTERVAL_MS, NULL);
}

// ===== 修改：get_cur_patient_id_name1 末尾添加 =====
// 添加：start_blink();  在函数末尾

// ===== 修改：change_to_next_patient =====
// 在更新 ui_Current0number0of0people0waiting0text0box 文本后
// 添加：start_blink();
```

### 注意事项

1. **非阻塞**：使用 `lv_timer_create`，不会阻塞主线程的 `lv_timer_handler()` 循环
2. **自动停止**：闪烁6次（3个周期约1.8秒）后自动恢复并停止
3. **重入安全**：如果闪烁正在进行中又触发新的闪烁，会先停止旧的再启动新的
4. **NULL安全**：定时器回调会检查对象是否存在
5. **程序启动时不闪烁**：main.c 中调用 `get_cur_patient_id_name1(NULL)` 时也会触发闪烁，如果不需要启动时闪烁，可以添加一个判断条件（例如检查事件参数 `e != NULL` 或使用条件标记）

### 可选增强

- 如果不想在程序启动时闪烁，可以在 `get_cur_patient_id_name1` 中判断 `e != NULL` 时才调用 `start_blink()`
- 可以添加音效配合闪烁（需要音频硬件支持）
- 可以将闪烁对象扩展到多个标签（如同时闪烁当前患者信息）
